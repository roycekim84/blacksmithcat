# 대장장이 고양이 - 전체 게임 기획서

## 1. 게임 개요

### 제목
대장장이 고양이

### 장르
세로형 모바일 캐주얼 강화 게임

### 핵심 감성
- 옛날 플래시 게임처럼 단순하고 즉각적인 재미
- 강화 버튼을 누르는 긴장감
- 성공했을 때 무기 외형이 바뀌는 보상감
- 실패했을 때 다시 도전하게 만드는 가벼운 손실감
- 픽셀아트 대장간의 따뜻하고 귀여운 분위기

### 핵심 문장
무기를 강화하고, 팔고, 재료를 모아 더 높은 강화에 도전하는 고양이 대장장이 게임.

## 2. 플레이어 목표

### 단기 목표
- 현재 무기를 더 높은 단계로 강화한다.
- 적당한 시점에 무기를 팔아 골드를 번다.
- 골드와 재료를 모아 다음 강화를 준비한다.

### 중기 목표
- +10 이후의 다른 강화 방식을 익힌다.
- 더 좋은 시작 무기, 보조 재료, 보호 아이템을 해금한다.
- 에피소드 1의 최고 단계인 +20 무기를 만든다.

### 장기 목표
- 전체 게임 기준 +99강 무기 달성.
- 에피소드별 새로운 강화 규칙, 무기군, 제작법, 대장간 확장 해금.

## 3. 핵심 루프

1. 기본 무기 또는 이전 단계 무기를 준비한다.
2. 강화에 필요한 골드와 재료를 확인한다.
3. 강화 버튼을 누른다.
4. 성공/실패 결과를 즉시 보여준다.
5. 무기 외형, 가치, 강화 수치가 갱신된다.
6. 계속 강화하거나 판매한다.
7. 판매 골드로 재료/보호권/시작권을 구입한다.
8. 더 높은 강화에 재도전한다.

## 4. 게임 톤

### 캐릭터
- 주인공은 신비한 능력을 가진 대장장이 고양이.
- 귀엽지만 장인 느낌이 있어야 한다.
- 망치질, 놀람, 기쁨, 좌절, 집중 등의 짧은 픽셀 애니메이션을 가진다.

### 세계관
- 작은 마법 대장간에서 시작한다.
- 고양이는 무기에 잠든 별빛/마력을 깨우는 능력을 가지고 있다.
- 강화 수치가 오를수록 무기는 평범한 검에서 신비한 마검처럼 변화한다.

### 비주얼
- 2D 픽셀아트.
- 세로형 모바일 화면에 맞춘 큼직한 버튼과 명확한 정보.
- 배경은 대장간: 화로, 모루, 선반, 재료 상자, 고양이.
- 무기는 화면 중앙에서 크게 보여준다.

## 5. MVP 범위

### 반드시 필요한 것
- 메인 강화 화면
- 무기 강화 기능
- 강화 성공/실패 연출
- 무기 판매 기능
- 골드/재료 보유량
- 상점
- 분해 기능
- +1 ~ +20 강화 테이블
- 저장/불러오기

### 나중에 추가 가능한 것
- 일일 퀘스트
- 도감
- 대장간 꾸미기
- 고양이 의상
- 에피소드 2 이후 무기군
- 의뢰 시스템
- 오프라인 보상

## 6. 에피소드 구조

### 에피소드 1: 작은 대장간의 첫 검
- 범위: +1 ~ +20
- 무기군: 검 중심
- 핵심 학습: 강화, 실패, 판매, 재료, 분해
- +10 이후부터 특수 강화 도입

### 이후 확장 예시
- 에피소드 2: 사막 상인의 곡도 / +21 ~ +40
- 에피소드 3: 얼음 왕국의 창 / +41 ~ +60
- 에피소드 4: 용광로의 대검 / +61 ~ +80
- 에피소드 5: 별빛 고양이의 전설무기 / +81 ~ +99

## 7. 재미의 핵심

### 눌렀을 때 바로 결과가 나와야 한다
강화 게임의 핵심은 버튼을 누르는 즉시 결과가 나오는 리듬이다. 결과 연출은 짧고 선명해야 하며, 반복 플레이를 방해하면 안 된다.

### 손실은 있지만 너무 화나면 안 된다
모바일 캐주얼 게임이므로 초반에는 파손을 강하게 쓰지 않는다. +10 이전은 실패해도 재료 손실 중심, +10 이후부터 하락/파손/보호권을 조심스럽게 도입한다.

### 팔기 타이밍이 선택이어야 한다
무조건 끝까지 강화하는 게임이 아니라, +7에서 팔지, +10까지 도전할지, +15를 노릴지 선택하게 만든다.

### 외형 변화가 중요하다
강화 수치가 오르면 수치만 바뀌는 것이 아니라 무기 색, 이펙트, 형태가 바뀌어야 한다.

## 8. 개발 원칙

- 첫 버전은 작게 만든다.
- 시스템은 데이터 테이블 기반으로 만든다.
- 강화 확률, 비용, 판매가, 재료 요구량은 코드에 하드코딩하지 않는다.
- 화면은 5개 내외로 시작한다.
- 광고/BM은 게임성을 해치지 않게 선택형으로만 붙인다.
